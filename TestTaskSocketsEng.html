This task should be a client-server program ready for MS Windows 7/8 64-bit. Work should be done as a solution for MS Visual Studio 2008/2010/2012 (it can be Express Edition).

Main task:

The application should be divided on two parts, server and client:

  1. Server should work with XML file that should contain pair tag – value. Server should read the pairs, add new pairs and update the existing values.
  2. Client should be a console application. It should connect to Server using Sockets, send requests and receive answers from Server. User should have possibility to use console commands to get access to XML file. You need to implement at least three commands: add <tag_name> <tag_value>, get <tag_name>, set <tag_name> <tag_value>.

Advanced task:

You can increase your chance to get this job if you solve these issues:

  1. Each console command should reply its status into console window.
  2. Design server part as a Windows Service.
  3. Cover your code by unit tests (example, Google Test).

Additional requirements:

  1. Source control systems (git, subversion, mercurial, etc) are very useful for collaboration work. Use one of it to promote your solution (GitHub, GoogleCode, etc).
  2. Use plain C++ and Win API, please, do not use Qt framework.
  3. Use STL (std::vector, std::map, etc) as much as possible.
  4. It is very good if you implement XML parser (it is not need to cover all XML standard, you need just parse tag – value pairs) but you can use any well-known libraries (example, TinyXML).
  5. C-style code is not welcome.
